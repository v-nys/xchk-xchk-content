Schriftelijk materiaal

Specifieke controleprocedures zijn geen intrinsiek onderdeel van het systeem. Iedere programmeur kan er zelf schrijven en deze beschikbaar maken voor collega's. Elke stukje code dat een "ja" of "nee" kan geven over een inzending, is denkbaar. Voorbeelden:

  • Is ingezonden code geldige syntaxis voor C♯?
  • Is ingezonden C♯-code equivalent aan een gekende oplossing?
  • Levert ingezonden MySQL-code zelfde data als modeloplossing?
  • Heeft een ingezonden elektriciteitsschema (voorgesteld als tekst) de gewenste karakteristieken?

Deze controles kunnen in geval van een ongewenste uitkomst verklaren waarom ze een "ja" of een "nee" hebben opgeleverd. Dus de auteur kan ook feedback bundelen die vanzelf gegeven wordt bij specifieke fouten. Deze feedback hoeft niet zuiver tekstueel te zijn.
